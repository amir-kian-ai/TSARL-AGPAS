# Quick Public Release Guide

This folder is prepared for a **public, non-reproducible research preview**.

Recommended steps:
1. Publish this folder as the public repository.
2. Keep the full private repository separate and offline or in a private remote.
3. State clearly in `README.md` and `NOTICE.md` that the full code will be released after paper publication.
4. Do not upload checkpoints, full logs, raw private notes, datasets, or final training scripts to the public repository.
5. Use the public repository to establish authorship timestamp, project structure, and visible research ownership without exposing the complete implementation.
